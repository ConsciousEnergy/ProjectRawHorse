## üìù Documentation Standards

### File Naming
- Use UPPERCASE for important files (PRD.md, README.md)
- Use descriptive names (INSTALLATION.md not INSTALL.md)
- Use underscores for multi-word files (DATA_ORGANIZATION.md)

### Document Structure
All documents should include:
1. Title and metadata (date, status, version)
2. Table of contents for long documents
3. Clear section headers
4. Code examples where relevant
5. Links to related documents

### Markdown Conventions
- Use `##` for main sections
- Use `###` for subsections
- Use code blocks with language tags
- Use tables for structured data
- Include emojis for visual navigation

## ü§ù Contributing to Documentation

When adding new documentation:

1. **Choose the right category:**
   - setup/ - Installation and configuration
   - development/ - Technical implementation
   - design/ - Visual and UX decisions
   - data/ - Data structure and sources

2. **Use the template:**
   ```markdown
   # Document Title
   
   **Date:** YYYY-MM-DD
   **Status:** Active/Draft/Archived
   **Category:** Setup/Development/Design/Data
   
   ## Summary
   Brief description...
   
   ## Content
   Main content...
   ```

3. **Update this README:**
   - Add entry to relevant table
   - Update "Recent Updates" section

4. **Cross-reference:**
   - Link to related documents
   - Keep links up to date
